1.1 问题：vue3中使用echarts图表时，报读取不到type的错 ​

在vue3中使用echarts图表时，原本使用来绑定使用，只要拉动时间轴或者切换就会报错，读取不到type， 这是因为echarts被代理成为响应式对象了，影响echarts对内部属性的访问，就会导致图表无法正确显示。

解决方案：使用普通变量声明echarts对象实例

//接收值并生成echarts
setEchart(val){
   const myEcharts= echarts.init(document.getElementById("echarts的ID名称"))
   myEcharts.setOption(val,true)
}
1.2 需求：一个echarts图表分成几份小图表展示，并加上相对应的背景图 ​

一个echarts图表分成几份小图表可以使用 ：根据分出的图表在xAxisS、yAxisS属性中设置gridIndex，

//arr是分成多少个小图表的数量
 for (let i=0; i<arr;i++){
    <!-- 给图表增加背景图片 -->
        graphic.push({
          type: "image", // 图形元素类型
          left:"center",
          top:"center",
          gridIndex:i, //这里就是关键属性
          style: {
            image: "https://images.669pic.com/element2/f3d9034e536af1592da76fe250108c0f.jpg", // 这里一定要注意、注意，必须是https开头的图片路径地址
            height: 260,
            width: widthEcharts,
            opacity: .5,
           display:"flex"
          }
        })
        xAxisS.push({
          type: "category",
          data: xAxis,//x轴的数据展示
          gridIndex:i,  //这里就是关键属性
          boundaryGap: ['20%', '50%'] ,// 设置 y 轴两边留白为 20%
        })
        yAxisS.push({
          type: "value",
          name:nameArr[i],
          gridIndex:i,
          axisLabel: {
            formatter: "{value}",
          },
        })
        }

上面的代码中增加了图表的背景图，但是位置问题就需要处理,在这里我使用的就是 来确定位置，用的笨方法，先固定echarts的图标位置，所以在增加背景图的时候获取到图表的x、y轴就可以了

 // 动态生成图片
    moveImage(val,arr){

1.3需求：在图表中设置一条预警线，超出预警线部分数据进行改变颜色 ​

目前的需求是线条超出阈值的部分变成红色，这里使用到的属性,使用分段式piecewise，

里面有个重要的属性，这个是要和数据的数量对应上的， gte：大于等于，lte：小于等于，gt：大于，lt：小于

<!-- 定义的颜色 -->
let color=[ "#5470c6","#91cc75","#fac858","#ee6666","#73c0de","#3ba272","#fc8452","#9a60b4","#ea7ccc"  ]
      <!-- 阈值 -->
 let yz=[1,10,23,45,56,67,45,34]
//根据数据的长度判断增加visualMap，
  for (let i=0;i<数据.length;i++){
  visualMap.push(
              {
                type: 'piecewise',
                show: false,
                seriesIndex: i, //这个是关键属性  
                pieces: [
                  {
                    gte:0,
